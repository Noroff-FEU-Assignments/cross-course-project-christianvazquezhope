Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

When delivering websites to customers it's important to keep security in mind.

Here are a few tips:

1: Always use SSL
SSL stands for Security Sockey Layer. SSL secures the information from being tampered with between the server and the user. Users can verify that our website is safe by looking for a lock in the address bar.

SSL also makes websites rank better in search engines.

2: Block SQL injections
Attackers can gain access to our website by manipulating our SQL databases by taking advantage of URL parameters. To prevent these injections we have to parameterize our querys in the database.

SQL injections attack the website or application running on our server.

3: Beware of cross-site scripting
Cross-site scripting, or XSS, can be injected into input forms, cookies, search fields, comments etc.. By taking advantage of these input fields attackers can get access to personal information like card numbers and passwords.

XSS attacks target the user of our website.

4: Watch out for cross-site request forgery
Cross-site request forgery, or CSRF, happen when attackers create malicious links that can be executed without the users knowledge. It is necessary to take security measures that detects unauthorized requests.

5: Only use known third party applications
Websites today are often built on frameworks and third party plugins and applications. To make sure our website is secure we should only implement third party applications that are widely tested and well known.
